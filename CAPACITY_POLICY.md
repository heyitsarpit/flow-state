# Capacity policy

[Back to the plan tracker](./TASK.md)

Authority: P0.6 records measured baselines and default/max policies before
implementation packets choose data structures. Budgets are not performance
claims; they are safety limits with typed diagnostics.

Measurements come from `pnpm --filter flow-state p0-1c:measure` on
2026-07-12. The adversarial tier uses 1,024 operations and key depth 12; P5.4
reruns the same command and compares ratios.

| Structure                    | Owner                         | Unit                                | P0.1c baseline                                                         | Default / max                                             | Overflow or eviction                                                              | Active-entry protection                                                  | Proof command                            |
| ---------------------------- | ----------------------------- | ----------------------------------- | ---------------------------------------------------------------------- | --------------------------------------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------------------------ | ---------------------------------------- |
| ResourceStore records        | ResourceStore                 | canonical resource instances        | 1,024 dehydrated resources in 21.470ms median                          | default 10,000 / max configurable                         | reject with `capacity.resource.records` unless caller requested explicit eviction | never evict fetching, subscribed, previewed, or actor-referenced entries | `pnpm --filter flow-state p0-1c:measure` |

| CLI output/evidence export   | CLI adapter                   | bytes/rows                          | evidence JSON bytes from P0.1c                                         | default 4MiB text, 16MiB JSON / max flag                  | fail closed with `capacity.cli.output` and truncation hint                        | semantic state already committed; export can fail                        | P4D.2/P5.2 CLI output test               |

Silent drop is never the default. A packet that wants a different default must
change this file and its proof command in the same commit.
