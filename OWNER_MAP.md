## Adapter routing

- React adapters under `react/**` should stay render/subscribe wrappers over a
  provided runtime. `use-actor.ts` and `use-resource.ts` must not own actor or
  resource semantics; P4B.1b/P4B.2 own remaining lifecycle and inference work.
- Server adapter `runtime/request-runtime.ts` creates/disposes a runtime for a
  host callback; P4C.2 owns request-scope finalization and isolation.
- CLI entry `cli/index.ts` may own Node argument parsing, file loading, and
  `ManagedRuntime` for CLI execution. It should project inspect/testing outputs,
  not implement runtime semantics.
- Inspection APIs under `inspect.ts` and `core/inspection/**` are projection and
  proof surfaces. They may walk definitions and facts but must not invoke client
  callbacks or claim runtime evidence without production facts.
