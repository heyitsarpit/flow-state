# Flow State [Planning interface for an Effect-native frontend runtime]

Flow State is being designed as a machine-first frontend runtime. The current work is not implementation; it is API design through planning, reference writing, and examples.

## Current Focus

- Keep all user-authored async work in Effect.
- Make workflow states explicit.
- Attach queries, mutations, streams, and effects to state boundaries.
- Use examples to discover the public API before implementing internals.
